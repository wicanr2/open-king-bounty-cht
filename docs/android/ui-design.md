# Android 觸控界面設計

御封戰將原本是鍵盤操作的 DOS/SDL 遊戲。要搬上 Android,最大的工作不是編譯,而是**把鍵盤互動轉成手指觸控**。這份文件先把界面設計定下來,實作(SDL2 android-project、NDK、資產打包)依此進行。

## 設計依據:引擎已經知道「這個畫面能按哪些鍵」

openkb 每個畫面都用一張 **keymap 表**描述當下有效的按鍵,再交給 `KB_event()` 等待輸入。例如難度選擇是上/下/Enter;城鎮選單是 `A`–`E`;Y/N 提問是 `y`/`n`。

```c
/* 城鎮選單的 keymap(節錄自 game.c) */
{ _NON, SDLK_a, 0, 0 },   /* A) 接任務   */
{ _NON, SDLK_b, 0, 0 },   /* B) 租船     */
{ _NON, SDLK_c, 0, 0 },   /* C) 情報     */
...
{ __NON, {90}, SDLK_SYN, 0, KFLAG_TIMER },  /* 重畫計時器 */
```

**這代表觸控層不用硬猜**:每進一個畫面,讀當前 keymap → 只顯示「這個畫面真的能按」的按鈕。選單畫面就浮出 A–E,地圖畫面就浮出方向鍵,Y/N 畫面就浮出兩顆「是/否」。畫面切換時控制列自動跟著換,不會一堆無用按鈕擋畫面。

觸控的本質就是:**手指事件 → 合成一個 `SDLK_*` → 餵進 `KB_event()`**,引擎其餘邏輯完全不動。

## 遊戲實際用到的輸入(決定要做哪些控制)

| 類別 | 按鍵 | 用在哪 |
|---|---|---|
| 方向 | ↑ ↓ ← → | 地圖移動、清單上下、戰鬥移動 |
| 確認 / 取消 | Enter、Space / ESC | 選定、繼續 / 返回、取消 |
| 字母快捷 | A–Z(常見 a–e、u、v…) | 選單項目(接任務/租船/用魔法…)、選角 A–D |
| 數字 | 0–9 | 招募數量、難度、數量輸入 |
| 清單捲動 | PageUp/Down、Home/End | 長清單(法術、城堡列表) |
| 是非 | Y / N | 退出、放棄等確認 |
| 文字 | 字母 | 角色命名 |
| 系統 | F8 / F9 / F10 / F12 | 切美術 / 切音樂 / 退出存檔 / 作弊 |

## 控制方案(分層)

設計原則:**常用的直接在拇指範圍內;少用的收進選單;選單項目盡量「直接點畫面」**。

1. **方向控制 — 左下虛擬 D-pad**
   兼任地圖移動、清單上下、戰鬥走位。也支援**滑動(swipe)**當作方向,玩家可二選一。
2. **A / B — 右下兩顆主鈕**
   A = Enter/Space(確認/繼續);B = ESC(返回/取消)。這兩顆 + D-pad 就能跑完所有「方向+確認」型畫面。
3. **情境快捷列 — 依當前 keymap 動態浮出**
   選單畫面把該畫面的字母選項做成**有標籤的按鈕**(讀 keymap + 畫面文字),例:城鎮顯示 `[A 接任務][B 租船][C 情報][D 造橋][E 買攻城]`。點按 = 送出該字母。這是最貼近 KB 又最好點的做法。
4. **直接點畫面項目**
   選單是文字清單時,**直接點那一行**就等於選它(把點到的 y 對應到選單列 → 該字母/該列)。情境快捷列是它的備援與顯式版。
5. **數字步進器 — 數量輸入時才出現**
   招募數量這種「輸入數字再 Enter」的畫面,浮出 `[−] 123 [+]` + `[最大][OK]`,免叫全鍵盤。
6. **角色命名 — 用 Android 系統輸入法(IME)**
   命名畫面呼叫 `SDL_StartTextInput()`,直接用手機原生鍵盤打字,送回引擎的命名緩衝。
7. **系統選單 — 右上 ☰**
   把不常用的 F8(切美術)/ F9(切音樂)/ 存檔離開 / 讀檔 / 退出 收進這顆,平常不佔畫面。
8. **清單捲動**
   長清單在快捷列旁出現 `[▲][▼]`(對應 PageUp/Down);或 D-pad 長按。

## 兩種輸入手感(玩家可切)

- **點擊模式(預設,適合選單/城鎮)**:盡量直接點畫面項目 + 情境快捷列;D-pad/A/B 仍在但縮小。
- **手把模式(適合地圖移動/戰鬥)**:D-pad + A/B 放大置前,情境鈕縮到角落。

切換鈕放 ☰ 系統選單裡;也可依畫面型態自動偏好(地圖/戰鬥→手把,選單→點擊)。

## 視覺與人體工學

- 控制元件**半透明**(預設約 60%),長按可暫時淡出看畫面;透明度可調。
- 觸控目標**最小 48dp**(Android 建議),拇指熱區優先。
- 支援左右手:D-pad 與 A/B 可左右對調。
- 直向(portrait):畫布置上、控制置下全寬;橫向(landscape):畫布置中、控制疊兩側(主要設計)。
- 安全區避開瀏海/手勢條(WindowInsets)。

## 與引擎的接點(實作時遵循,本階段先定義)

1. **觸控 → 鍵**:Android 觸控事件在 SDL 層轉成合成 `SDL_KEYDOWN(SDLK_*)`,進既有 `KB_event()`;引擎主流程零改動。
2. **情境快捷列資料來源**:`KB_event` 收的 keymap 已含「本畫面有效鍵」清單 → 導出給觸控層渲染(需在 keymap 旁補一個「可給觸控顯示的標籤」欄位,或由觸控層對照畫面字串)。
3. **命名**:命名畫面開關 `SDL_StartTextInput()/StopTextInput()`,吃 `SDL_TEXTINPUT`。
4. **存檔路徑**:Android 內部儲存(`getFilesDir()` / SDL `SDL_AndroidGetInternalStoragePath()`),不可用相對 cwd(唯讀)。
5. **資產**:`data/free` + `cjk24.bin` 打包進 APK 的 `assets/`,首次啟動解到內部儲存或用 SDL_RWops 直讀。
